Primeros pasos

  1. Inicia el servidor Backend
    cd Backend && npm start
  2. Abre Swagger UI en tu navegador
    http://localhost:3000/api-docs
  3. Obtén un token JWT haciendo login
    Endpoint: POST /api/v1/auth/login
  4. Autoriza en Swagger
    Haz clic en "Authorize" e ingresa tu token
  5. ¡Prueba los endpoints!
    Haz clic en cualquier endpoint y selecciona "Try it out"

Autenticación JWT

✅ Todos los endpoints protegidos requieren autenticación

Pasos para autenticar:

  1. Realizar login
    POST /api/v1/auth/login
    Body: {"email":"tu_email@ferreteria.com","password":"password"}
  2. Copiar el token JWT de la respuesta
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  3. En Swagger: Haz clic en "Authorize"
    Ingresa: Bearer YOUR_JWT_TOKEN
  4. En cURL: Agrega el header
    -H "Authorization: Bearer YOUR_JWT_TOKEN"

Ejemplo de uso con cURL

1. Login

curl -X POST http://localhost:3000/api/v1/auth/login \
-H "Content-Type: application/json" \
-d '{"email":"juan@ferreteria.com","password":"password123"}'

2. Listar productos (sin autenticación)

curl http://localhost:3000/api/v1/productos

3. Obtener info del usuario (con autenticación)

curl -H "Authorization: Bearer YOUR_TOKEN" \
http://localhost:3000/api/v1/auth/me

4. Registrar venta

curl -X POST http://localhost:3000/api/v1/ventas \
-H "Content-Type: application/json" \
-H "Authorization: Bearer YOUR_TOKEN" \
-d '{
"lineas": [
{"id_producto": 1, "cantidad": 2, "precio_unitario": 99.99}
],
"metodo_pago": "efectivo"
}'

Problemas comunes:

  • Swagger no aparece - Reinicia el servidor y borra el caché
  • Token inválido - Obtén uno nuevo en /api/v1/auth/login
  • CORS error - Verifica que CORS esté habilitado en app.js
  • 404 Not Found - Verifica la URL y que el servidor esté corriendo
